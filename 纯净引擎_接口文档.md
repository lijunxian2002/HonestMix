## 一、项目结构

```
Source/
├── PluginProcessor.*      ← 信号链入口（UI 只通过此类接口操作）
├── PluginEditor.*         ← 空白桩（UI 层替换本文件填肉）
├── Core/
│   ├── Design.h           ← 设计令牌（颜色/字体/圆角/间距 —— UI 层直接抄）
│   ├── AppState.*         ← 状态中心（ValueTree 持久化）
│   └── HeadphoneDatabase.*← 1603 副耳机 + 曲线/声卡表 + FIR 指针
├── DSP/
│   ├── CorrectionEngine.* ← FIR 卷积（juce::Convolution, 1024 taps, 始终 ON）
│   ├── WetDryMixer.*      ← 翻译度 0~200 混合（SIMD，atomic 线程安全）
│   └── fir_data.h         ← 25MB AutoEq 系数表（硬链接 → 数据中心仓库）
└── Net/
    └── FeedbackClient.*   ← Supabase 反馈（后台线程，无阻塞）
```

## 二、UI 调用的全部接口（PluginProcessor 公共方法）

### 2.1 翻译度

```cpp
// 读（音频线程 / UI 线程均可）
float getTranslationDegree() const noexcept;   // 0~200

// 写（UI 线程调用，内部走 atomic 同步音频线程）
void  setTranslationDegree (float v);         // 自动夹取 0~200
```

### 2.2 耳机

```cpp
// 获取引擎引用（只读）
CorrectionEngine&        getCorrectionEngine() noexcept;
const HeadphoneDatabase& getDatabase() const noexcept;

// 换耳机（调用方负责 suspendProcessing(true)/false 包裹）
void setProfileIndex (int idx);              // kProfiles 下标
void setProfileByName (const juce::String& name);

// 当前耳机全名
juce::String getProfileName() const;
```

### 2.3 宿主 BPM

```cpp
float getHostBpm() const noexcept;   // atomic，音频线程写入 → UI 读
```

### 2.4 监听模式

```cpp
void       setListenMode (ListenMode m);      // ListenNormal / ListenMono / ListeniPhone / ListenCar
ListenMode getListenMode() const noexcept;
```

### 2.5 引导 / 首次流程

```cpp
bool isOnboarded() const noexcept;   // 是否已完成首次设置
void setOnboarded();                 // 标记完成
```

### 2.6 状态持久化（DAW 工程存取）

```cpp
AppState& getAppState() noexcept;

// AppState 暴露的领域字段：
//   appState.getTranslationDegree()      → float 0~200
//   appState.getProfileName()            → String (全名)
//   appState.isOnboarded()               → bool
//   appState.getListenMode()             → int
//   appState.getCurveIndex()             → int  (0~3)
//   appState.getInterfaceIndex()         → int  (0~7)
//   appState.getMixHabitMinutes()        → int  (default 60)
```

## 五、信号链（processBlock 内部顺序）

```
① mixer_.captureDry(buffer)          // 预存干信号
② correctionEngine_.process(buffer)  // FIR 卷积（始终 ON）
③ mixer_.mixInPlace(buffer)          // 翻译度混合: out = wet·t + dry·(1-t), t=degree/100
④ ListenMode 模拟（可选）            // Mono 合成 / iPhone 高通 / Car 低架
⑤ hostBpm 跟读（从宿主 PlayHead 取）
```

## 六、UI 层编辑须知

| 文件 | 你做什么 |
|:---|:---|
| `PluginEditor.h/.cpp` | **你改这里**——放 Layout 尺寸、所有组件声明与绘制 |
| `PluginProcessor.h` | **不看**——API 锁定 |
| `Core/Design.h` | **改颜色/字体常量**——改完整个 UI 联动变色 |
| `Core/AppState.h` | **如需新持久字段**——加 setter/getter（自动序列化） |
| `CMakeLists.txt` | **新增 .h/.cpp 时**——在 target_sources 区加两行 |
