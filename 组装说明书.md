## 三、布局规格（resized() 的逐行实现）

### 3.1 整体结构

```
PluginEditor 总尺寸：600 × 328（设计基准）

内部结构（从上到下）：
┌────────────────────────────────────────────┐
│ padding: 22(top) 20(right) 16(bottom) 20(left) │
│                                             │
│ TopBar          高度: auto（约 19px）        │
│   margin-bottom: 12px                       │
│                                             │
│ MainGrid        高度: 剩余空间减去下方区域     │
│   ├─ Cockpit    flex: 1（自适应）            │
│   └─ Fader      width: 120px（固定）         │
│   gap: 18px                                 │
│                                             │
│   margin-bottom: 10px                       │
│                                             │
│ InfoRow         高度: ~50px                 │
│   margin-bottom: 10px                       │
│                                             │
│ BottomRow       高度: ~28px                 │
│   padding-top: 9px                          │
│   border-top: 1px                           │
└────────────────────────────────────────────┘
```

### 3.2 伪代码

```
resized():
  b = getLocalBounds().reduced(22, 20, 16, 22)   // 上 右 下 左

  // --- TopBar ---
  topBarHeight = 19  // 9px 字体 + 9px padding-bottom + 1px border
  topBar.setBounds(b.removeFromTop(topBarHeight))
  b.removeFromTop(12)   // margin-bottom

  // --- InfoRow（先算高度，因为 MainGrid 高度 = 剩余 - InfoRow - BottomRow）---
  infoRowHeight = 50
  bottomRowHeight = 28
  mainGridHeight = b.getHeight() - infoRowHeight - 10 - bottomRowHeight - 10

  // --- MainGrid ---
  mainGridBounds = b.removeFromTop(mainGridHeight)
  cockpitBounds = mainGridBounds.removeFromLeft(mainGridBounds.getWidth() - 120 - 18)
  faderBounds = mainGridBounds.removeFromRight(120)
  // 中间 gap 18px 自动被 removeFromLeft/Right 吃掉
  cockpit.setBounds(cockpitBounds)
  fader.setBounds(faderBounds)

  b.removeFromTop(10)   // margin

  // --- InfoRow ---
  infoRow.setBounds(b.removeFromTop(infoRowHeight))

  b.removeFromTop(10)   // margin

  // --- BottomRow ---
  bottomRow.setBounds(b.removeFromTop(bottomRowHeight))

  // --- Overlays（覆盖整个插件区域，初始隐藏）---
  overlayBounds = getLocalBounds()
  overlayBPM.setBounds(overlayBounds)
  overlayMonitor.setBounds(overlayBounds)
  overlayFeedback.setBounds(overlayBounds)

  // --- Tooltip（由 JS/鼠标事件动态定位，resized 中不设 bounds）---
```

### 3.3 Cockpit 内部布局

```
Cockpit::resized():
  b = getLocalBounds().reduced(8)  // cockpit padding

  // VUPanel（顶部）
  vuHeight = 22  // 4px 轨道 + 6px padding×2 + 2px border×2 + 2px margin
  vuPanel.setBounds(b.removeFromTop(vuHeight))
  b.removeFromTop(6)   // margin-bottom

  // 仪表盘区域（剩余高度）
  dashboardBounds = b

  // 仪表盘子组件全部铺满 dashboardBounds
  curveGrid.setBounds(dashboardBounds)
  curveLabels.setBounds(dashboardBounds)
  curveCanvas.setBounds(dashboardBounds.reduced(1))  // inset: 1px
  starField.setBounds(dashboardBounds.expanded(10))  // overflow: -10px

  // 呼吸灯（绝对定位，右上角）
  breath.setBounds(dashboardBounds.getRight() - 14 - 3,
                   dashboardBounds.getY() + 12,
                   3, 3)
```

### 3.4 InfoRow 内部布局

```
InfoRow::resized():
  b = getLocalBounds()
  width = b.getWidth()
  chipWidth = (width - 12) / 3   // 3 列，gap 6px×2 = 12px

  headphoneChip.setBounds(0, 0, chipWidth, b.getHeight())
  curveChip.setBounds(chipWidth + 6, 0, chipWidth, b.getHeight())
  interfaceChip.setBounds((chipWidth + 6) * 2, 0, chipWidth, b.getHeight())
```

### 3.5 BottomRow 内部布局

```
BottomRow::resized():
  b = getLocalBounds().reduced(0, 9, 0, 0)  // padding-top: 9px
  btnWidth = (b.getWidth() - 16) / 3         // 3 列，gap 8px×2 = 16px

  bpmBtn.setBounds(0, 0, btnWidth, b.getHeight())
  monitorBtn.setBounds(btnWidth + 8, 0, btnWidth, b.getHeight())
  feedbackBtn.setBounds((btnWidth + 8) * 2, 0, btnWidth, b.getHeight())
```

## 四、信号连接（事件流）

### 4.1 核心数据流

```
                    ┌──────────────┐
                    │ PluginProces-│  ← 引擎（音频线程 + 数据库）
                    │ sor          │
                    └──┬───┬───┬──┘
                       │   │   │
        ┌──────────────┼───┼───┼──────────────┐
        │              │   │   │              │
        ▼              ▼   ▼   ▼              ▼
   ┌─────────┐   ┌──────────────────────┐  ┌──────────┐
   │  Fader  │   │    Dashboard         │  │ InfoRow  │
   │ (推子)   │   │  (CurveCanvas+Grid)  │  │ (Chips)  │
   └────┬────┘   └──────────┬───────────┘  └────┬─────┘
        │                   │                   │
        │  翻译度改变        │  耳机/曲线/声卡改变  │
        ▼                   ▼                   ▼
   ┌────────────────────────────────────────────────┐
   │               PluginProcessor                    │
   │  setTranslationDegree()  /  setProfileIndex()   │
   │  setCurveIndex()  /  setInterfaceIndex()         │
   └────────────────────────────────────────────────┘
```

### 4.2 逐条连接

```
连接 1：Fader → 多个目标
  触发：用户拖动推子
  数据：degree（float, 0~200）
  
  目标 A：VUPanel.setLevel(degree / 200.0f)
    → VU 填充条宽度 + 颜色变化
  
  目标 B：CurveCanvas.setTranslationDegree(degree)
    → 矫正曲线重新计算 Y 坐标 + 重绘
  
  目标 C：stDot（TopBar 内）颜色更新
    → 状态灯 = spectrumColor(degree / 200.0f)
  
  目标 D：PluginProcessor.setTranslationDegree(degree)
    → 同步到音频线程，改变 Wet/Dry 混合比

连接 2：HeadphoneChip → 多个目标
  触发：用户在下拉菜单中选中耳机
  数据：profileIndex（int, 0~1602）
  
  目标 A：PluginProcessor.setProfileIndex(index)
    → 加载对应 FIR 到 CorrectionEngine
  
  目标 B：CurveCanvas.loadProfile(index)
    → FIR → FFT → rawYs → 重绘原始曲线 + 矫正曲线
  
  目标 C：Tooltip 标签更新为耳机名

连接 3：CurveChip → CurveCanvas
  触发：用户选择目标曲线
  数据：curveIndex（int, 0~3）
  
  目标：CurveCanvas.setTargetCurve(index)
    → 重新计算目标线位置 + 矫正曲线

连接 4：InterfaceChip → PluginProcessor
  触发：用户选择声卡
  数据：interfaceIndex（int, 0~7）
  
  目标：PluginProcessor 存储（当前版本可能不改变音频处理）

连接 5：BottomRow 按钮 → Overlay
  触发：点击按钮
  bpmBtn.onClick     → overlayBPM.setVisible(true)
  monitorBtn.onClick → overlayMonitor.setVisible(true)
  feedbackBtn.onClick → overlayFeedback.setVisible(true)

连接 6：Overlay 关闭按钮 → 隐藏自身
  overlayBPM.closeBtn.onClick     → overlayBPM.setVisible(false)
  overlayMonitor.closeBtn.onClick → overlayMonitor.setVisible(false)
  overlayFeedback.closeBtn.onClick → overlayFeedback.setVisible(false)

连接 7：BPM Overlay 内部
  BPM ± 按钮       → 更新 BPM 值 + 重新计算延迟表
  TAP TEMPO 按钮    → 记录时间戳 + 计算 BPM + 更新延迟表

连接 8：仪表盘交互
  CurveCanvas.onHover(pos) → Tooltip.show(x, y, freq, db, mode)
  CurveCanvas.onLeave()    → Tooltip.hide()
  CurveCanvas.onClick()    → 循环切换视图模式（默认→原始→目标→默认）
```

## 五、Z 轴层级（绘制顺序）

```
从底到顶：

1. PluginEditor 背景渐变（最低层）
   ├── 氛围光（径向渐变叠加）
   └── 装饰线（中心竖线 + 两条横线）

2. Cockpit（驾驶舱）
   ├── 铆钉 + 船体撑角 + 内装饰线
   ├── VUPanel
   ├── StarField          ← z=0（星空在网格下面）
   ├── CurveGrid          ← z=1
   ├── CurveCanvas        ← z=2（曲线在星空上面）
   ├── CurveLabels        ← z=3（标签在最上面）
   └── Breath             ← z=5（呼吸灯在一切之上）

3. Fader（推子）

4. InfoRow（Chip 行）
   └── Chip dropdowns     ← z=9999（下拉菜单必须覆盖一切）

5. BottomRow（按钮行）

6. Tooltip               ← z-index 等效：最顶层浮层

7. Overlays（弹窗）       ← 覆盖整个插件，打开时在最上面
```

## 六、组装顺序（建议）

```
阶段 1：空壳跑通（0.5 天）
  ├── 创建所有 14 个 .h/.cpp 空文件
  ├── 每个类的 paint() 只画一个占位矩形（不同颜色区分）
  ├── PluginEditor 按 §3 布局把所有组件放好
  └── 编译 → 能看到 14 个彩色方块各就各位 → ✅

阶段 2：静态组件填肉（3 天）
  ├── TopBar     → 填 paint()
  ├── BottomRow  → 填 paint()
  ├── InfoRow    → 填 paint() + Chip 下拉菜单逻辑
  ├── VUPanel    → 填 paint()
  ├── Breath     → 填 paint() + Timer 呼吸动画
  ├── Cockpit    → 填 paint()（铆钉 + 装饰线）
  └── Fader      → 填 paint() + 鼠标拖拽交互
  每完成一个 → 在组装层里看到灰色方块变精致组件

阶段 3：仪表盘填肉（3 天）
  ├── CurveGrid    → 填 paint()（网格线 + 零线虚线）
  ├── CurveLabels  → 填 paint()（15 个坐标标签 + hover 淡入）
  ├── CurveCanvas  → 填 paint()（3 组曲线 SVG → Path）+ 点击切换
  └── StarField    → 填 paint() + Timer 30Hz（45 颗星 + 追逐星 + 心形）

阶段 4：弹窗 + 提示框（1.5 天）
  ├── Tooltip       → 填 paint() + 显示/隐藏逻辑
  ├── OverlayBPM    → 填 paint() + BPM 逻辑 + 延迟表计算
  ├── OverlayMonitor→ 填 paint()
  └── OverlayFeedback→ 填 paint()

阶段 5：信号连接（1 天）
  └── 按 §4 逐条连接信号，全部 8 条接好

阶段 6：数据桥接（1 天）
  ├── 实现 §10.9 的 FIR → FFT → rawYs 路径
  ├── HeadphoneChip 选中 → 仪表盘加载真实曲线
  └── Fader 拖动 → 矫正曲线实时变形

阶段 7：调色 + 帧率优化 + DAW 测试（2 天）
```

**总组装时间：约 12 个工作日。**
